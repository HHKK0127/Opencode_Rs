## 🎯 プロジェクト概要

### 目的
TypeScript で実装された 43K行のOpenCodeツールを、Rust を使用した高パフォーマンスなハイブリッドバックエンドで置き換える。Strangler Fig パターンを採用し、段階的な移行を実現。

### アプローチ
**Strangler Fig パターン** - 既存システムを徐々に新システムで置き換える

フェーズ1: 認証・基盤層（JWT, middleware）
フェーズ2: ファイル処理（アップロード・検索）
フェーズ3: ストレージ移行（S3/MinIO）
フェーズ4(計画): マイクロサービス化・Redis統合
